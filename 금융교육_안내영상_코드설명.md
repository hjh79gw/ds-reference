# 금융교육 안내영상 컴포넌트 문서

## 개요

금융 멘토-멘티 교육 앱에서 사용되는 **30초 교육 안내 애니메이션** 컴포넌트입니다. 어르신(멘티)이 실습 시 개인정보 대신 연습용 정보를 사용하도록 안내하는 목적으로 제작되었습니다.

모바일 환경에 최적화되어 있으며, 세로 풀스크린 레이아웃, 큰 글씨, 터치 친화적 버튼을 적용했습니다.

---

## 기술 스택

- **React** (함수형 컴포넌트)
- **TypeScript** (.tsx)
- **CSS-in-JS** (인라인 스타일)
- **Web Speech API** (TTS 나레이션)
- **CSS @keyframes** (애니메이션)
- **SVG** (캐릭터 및 아이콘)

## 씬 구성

| 구간 | 씬 ID | 내용 | 핵심 메시지 |
|------|--------|------|-------------|
| 0~5초 | welcome | 캐릭터 인사 | "반갑습니다!" |
| 5~15초 | practice | 연습용 정보 안내 | "실제 정보 대신 홍길동, 1234 사용" |
| 15~25초 | protect | 개인정보 보호 강조 | "소중한 개인정보는 비밀!" |
| 25~30초 | start | 시작 유도 | "자, 시작해볼까요?" |

## 코드 구조 상세

### 1. 상수 정의

```tsx
// 씬 타임라인 정의
interface Scene {
  start: number;   // 시작 시간(초)
  end: number;     // 종료 시간(초)
  id: string;      // 씬 식별자
}

const SCENES: Scene[] = [
  { start: 0, end: 5, id: "welcome" },
  { start: 5, end: 15, id: "practice" },
  { start: 15, end: 25, id: "protect" },
  { start: 25, end: 30, id: "start" },
];

// 각 씬의 나레이션 텍스트 (TTS로 읽어줌)
const NARRATIONS: string[] = [ ... ];

// 씬 선택 버튼에 표시되는 짧은 라벨
const SUBTITLES: string[] = [ ... ];
```

### 2. 상태 관리

```tsx
const [time, setTime] = useState<number>(0);           // 현재 재생 시간 (0~30)
const [isPlaying, setIsPlaying] = useState<boolean>(false);  // 재생 중 여부
const [currentScene, setCurrentScene] = useState<number>(0); // 현재 씬 인덱스 (0~3)
const intervalRef = useRef<...>(null);                  // setInterval 참조
const lastSpokenScene = useRef<number>(-1);             // 마지막으로 나레이션 재생한 씬
```

### 3. 핵심 로직

#### 타이머 (재생 엔진)

```tsx
useEffect(() => {
  if (isPlaying) {
    intervalRef.current = setInterval(() => {
      setTime((prev) => {
        if (prev >= 30) {
          setIsPlaying(false);        // 30초 도달 시 정지
          window.speechSynthesis.cancel();
          return 30;
        }
        return prev + 0.05;           // 0.05초씩 증가
      });
    }, 60);  // 60ms 간격 = 약 1.2배 느린 속도
  }
  return () => {
    if (intervalRef.current) clearInterval(intervalRef.current);
  };
}, [isPlaying]);
```

- `setInterval`로 매 60ms마다 `time`을 0.05씩 증가
- 일반 속도(50ms)보다 느린 60ms 간격 → **1.2배 천천히 재생**
- 30초 도달 시 자동 정지

#### 씬 전환 감지

```tsx
useEffect(() => {
  const scene = SCENES.findIndex((s) => time >= s.start && time < s.end);
  if (scene !== -1) setCurrentScene(scene);
}, [time]);
```

- `time`이 변경될 때마다 현재 어떤 씬에 해당하는지 계산
- `SCENES` 배열에서 시간 범위가 맞는 씬을 찾아 `currentScene` 업데이트

#### 씬 진행도 계산

```tsx
const sceneProgress = SCENES[currentScene]
  ? (time - SCENES[currentScene].start) /
    (SCENES[currentScene].end - SCENES[currentScene].start)
  : 0;
```

- 현재 씬 내에서 얼마나 진행되었는지 0~1 사이 값으로 계산
- 이 값을 활용해 **씬 내 세부 애니메이션 트리거**
  - 예: `sceneProgress > 0.3`이면 실명 → 홍길동으로 전환

### 4. TTS 나레이션

#### 음성 재생 함수

```tsx
const speak = (text: string) => {
  window.speechSynthesis.cancel();  // 기존 음성 중단
  const utterance = new SpeechSynthesisUtterance(text);
  utterance.lang = "ko-KR";        // 한국어
  utterance.rate = 0.85;            // 느린 속도
  utterance.pitch = 1.0;
  utterance.volume = 1.0;

  // 한국어 음성 자동 선택
  const voices = window.speechSynthesis.getVoices();
  const koreanVoice = voices.find((v) => v.lang.startsWith("ko"));
  if (koreanVoice) utterance.voice = koreanVoice;

  window.speechSynthesis.speak(utterance);
};
```

#### 씬 변경 시 자동 재생

```tsx
useEffect(() => {
  if (isPlaying && currentScene !== lastSpokenScene.current) {
    lastSpokenScene.current = currentScene;
    speak(NARRATIONS[currentScene]);
  }
}, [currentScene, isPlaying]);
```

- `lastSpokenScene`으로 같은 씬의 나레이션이 중복 재생되지 않도록 방지
- 일시정지 후 재생 시 `lastSpokenScene`을 -1로 리셋해서 다시 재생되도록 처리

### 5. UI 구조 (모바일 최적화)

```
┌──────────────────────┐
│  상단바 (시간/타이틀)    │  ← 상태 표시
│  프로그레스바           │  ← 전체 진행률
│                      │
│                      │
│   메인 씬 컨텐츠        │  ← 씬별 애니메이션
│   (캐릭터, 정보 등)     │     세로 중앙 정렬
│                      │
│                      │
├──────────────────────┤
│  자막 (검은 배경)       │  ← 나레이션 텍스트 16px
├──────────────────────┤
│  씬 선택 버튼 4개       │  ← 터치로 씬 이동
│  [▶ 재생] [↺]         │  ← 최소 높이 54px
└──────────────────────┘
```

#### 모바일 대응 포인트

- `maxWidth: 430px` → 모바일 화면 너비에 맞춤
- `minHeight: 100dvh` → 모바일 브라우저 주소창 고려한 전체 높이
- 버튼 `minHeight: 54px` → 터치 영역 확보
- `-webkit-tap-highlight-color: transparent` → 터치 시 하이라이트 제거
- 세로 레이아웃으로 전체 구성

## 커스터마이징 가이드

### 나레이션 텍스트 변경

`NARRATIONS` 배열의 텍스트를 수정하면 자막과 TTS 모두 자동 변경됩니다.

```tsx
const NARRATIONS: string[] = [
  "여기에 새로운 나레이션 텍스트",
  ...
];
```

### 씬 시간 조정

`SCENES` 배열의 `start`/`end` 값을 변경하세요. 전체 시간도 함께 조정해야 합니다.

```tsx
// 타이머 종료 조건의 30을 원하는 값으로 변경
if (prev >= 40) {  // 40초로 변경
```

### TTS 속도 조정

```tsx
utterance.rate = 0.85;  // 0.1(매우 느림) ~ 2.0(매우 빠름)
```

### 색상 변경

`sceneBg` 변수에서 각 씬의 배경 그라데이션을 수정하세요.

## 주의사항

- **Web Speech API**는 브라우저마다 음성 품질이 다릅니다. 맥/iOS Safari가 비교적 자연스럽고, Chrome도 무난합니다.
- **모바일 TTS**: iOS Safari에서는 사용자 터치 이벤트 이후에만 TTS가 작동합니다. 재생 버튼을 터치하면 정상 동작합니다.
- 더 자연스러운 음성이 필요하면 네이버 CLOVA Voice, Google Cloud TTS 같은 외부 API를 연동할 수 있습니다.
- 실제 앱에 적용할 때는 `localStorage`로 "영상 시청 완료" 상태를 저장해서 최초 1회만 보여주는 로직을 추가하는 것을 권장합니다.
